Обработка исключений это место, где код будет исполняться и проверяться на исключения
Обработчик исключений есть всегда, даже атм где его не объявляли
Дефотлный обработчик - обработчик JVM который ловит исключения

1.TRY-CATCH
Это блок кода, который указывает на место исполнения кода, которое будет проверяться на исключения
Блоков catch может быть много, пишутся по иерархии исключений от низших с высшему
try {исполняющий код} catch (Exception e) {исполняющий код в случае возникновения исключения}

2.FINALLY
Помимо самого блока try-catch может быть добавлен блок с final
Это блок, который исполняется в любом случае после завершения обаботки исключения (обычно используется для закрытия потоков)
try {} catch() {} finally {}

3.TRY-CATCH-RESOURCES
Есть еще также блок try-catch с ресурсами (ресурс - поток)
Это блок который автоматически закрывает все открытые потоки в блоке ресурсов (ресурсы должны быть подписаны на интерфейс Closeable)
try (ресурсы) {исполняющий код} catch (Exception e) {исполняющий код в случае возникновения исключения}
