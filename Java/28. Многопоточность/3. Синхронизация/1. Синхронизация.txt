Синхронизация - координация выполнения нескольких потоков. Главным для синхронизации в Java является понятие монитора, контролирующего доступ к объекту
Монитор реализует принцип блокировки. Если объект заблокирован одним потоком, это означает, что этот поток захватил монитор объекта. В этот момент времени объект недоступен для других потоков

У каждого объекта в Java имеется свой монитор, то есть синхронизировать можно любой объект. Для поддержки синхронизации предусмотрено ключевое слово - synchronized

Нестатический метод:
synchronized - синхронизация нестатического метода. Говорит о том, что в единицу времени у этого объекта только один поток может быть вызван у методов этого объекта, помеченными как сихроннированными. Потоки, которые попытаются сделать параллельный вызов этих методов, перейдут в режим ожидания

При выполнении потоком синхронизированного метода, монитор блокирует доступ другим потокам ко всем синхронизированным методам этого объекта. В роли монитора выступает объект у которого вызываются эти самые методы. На обычные несинхронизированные методы блокировки не применяются

Параллельные потоки могут спокойно одновременно вызывать синхронизированные методы у двух разных объектов, так как мониторы будут тоже разными

Статическйи метод:
Это означает, что только один поток, в единицу времени, у данного класса, может вызвать только один статических синхронизированный метод. В роли монитора выступает сам класс

Примечания:
1. Синхронизация и переключение между потоками ресурсоемкая операция (время выполнения увеличивается и приложение начинает работать медленнее), поэтому использовать данные механизмы нужно осторожно, только там, где между потоками существует конкуренция за ресурсы
2. Проектирование многопоточного исполнения программ, может привести к Deadlock
