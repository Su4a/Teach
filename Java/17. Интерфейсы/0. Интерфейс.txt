Интерфейс
Интерфейс - класс (контракт), описывающий действия его использующих классов, нужен для публичности, публичных взаимосвязей API, выполнение контрактов (интерфейс не может быть объектом)
Свойства интерфейса могут быть только публичные, финальные и статичные
Классы, реализованные от интерфейса, должен реализовать методы интерфейса (контракт)
Контракт - определяет набор методов у класса публичных, которые используют контракторы
Количество интерфейсов у класса может быть неограниченное количество
Общий тип классов можно присвоить по имени интерфейса(полиморфизм)
Интерфейс служит для создания публичных свойств и поведений будущих объектов
Все методы в интерфейсе по умолчанию публичные и абстрактные. У наследников выставляются соответствующие модификаторы. Реализация с таким же модификатором
в имени интерфейса всегда ставится первой буквой заглавная (I)
Интерфейсы могут наследоваться друг от друга, приобретая методы и поля своих родителей
К интерфейсам можно применять множественное наследование
При одинаковых сигнаторах методов (с разными аргументами входов) интерфейсах происходит перегрузка этих методов у реализующего класса, если методы полностью одинаковые, тогда перегрузка происходит на уровне интерфейсов
В интерфейсе могут быть дефолтные методы (неабстрактные методы, которые имеют тело) - default
Дефолтный метод можно переопределить в классе
Если в двух интерфейсах есть 2 одинаковых дефолтных метода, они автоматически будут абстрактными
Могут быть приватные методы
Могут быть статические методы (статические методы не имплементируются на классы, вызываются непосредственно через интерфейс(напрямую))
Объект интерфейса создать нельзя

interface имя_интерфейса
class имя_класса implements имя_интерфейса

interface - объявление интерфейса
implements - присваивание классу интерфейс (реализовывает интерфейс)
В интерфейсе могут быть поля (статические и финализированные) и методы с телом (дефолт методы(default модификатор), без необходимости реализации его у наследников)

Подключение интерфейсов:
класс implements интерфейс -  2 и более через запятую

Класс, реализующий интерфейс, должен реализовать все его методы, если они не дефолтные
Дефолтный метод - метод интерфейса, который был реализован в самом интерфейсе. Классу реализовывать дефолтные методы не обязательно. Можно переопределить у класса

default void slorWun() {} - дефолтная реализация
