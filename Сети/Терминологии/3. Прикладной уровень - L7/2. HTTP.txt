ПРОТОКОЛ HTTP - HyperText Transfer Protocol
Это протокол передачи гипертекста

Для отображения WEB страничек на компьютерах, на серверах работают веб сервера (приложения), которые принимают запросы на определенные ресурсы и предоставляют контент

Пример:
Клиент обращается к веб серверу на запрос странички
Первым делом веб сервер создает сокет
Клиент передает запрос
Веб сервер парсит запрос и отдается страничка из директории

Методы HTTP:
- JSON - между клиентом и http методами
- HTTP - между http методами и самим сервером
Запросы, общение между веб сервером и клиентом, где веб сервер выступает в роли официанта между клиентом и директориями контента
Запросы: 
GET - запрос на получение контента
POST - запрос на изменение контента, отправление контента на веб сервер
PUT - запрос на положить файл на сервер
DELETE - запрос на удаление файлов с серверов
и др...

HTTP answer codes:
Задача веб сервера, прежде чем вернуть контект, еще вернуть и код ответа
1xx - информационный ответ
2xx - код успеха, контект существует, страничка существует и тд
3xx - перенаправление запроса на другой url
4xx - ошибка запроса от клиента, когда запрашиваемый контект не существует и тд
5xx - веб серверная ошибка

Порты по умолчанию
HTTP - 80 - нешифрвоанное соединение (данные в открытом виде)
HTTPS - 443 - шифрованное соединение (зашифрованные данные) по технологии TLS
Порты можно менять, но тогда и сообщить клиенту о том, что азпрос нужно отправлять на другой порт
TLS - транспорт лайрс секьюрити, проверяет сертификат на сайте, достает от туда открытый ключ, делает дификт хелман с помощтю этого открытого ключа с нашим сайтом, в результате которого получается сессионный ключ и уже на нем шифруются все даннные которые идут по HTTP (защищает от атак по середине, мидл атак)
