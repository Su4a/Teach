Докерфайл это файл по которому создается образ
alpine - самый минималистичный образ для сборки образов

Чтобы собрать образ из докерфайла:
docker build . - собрать образ, должны находиться в папке с докерфайлом
docker build -t <имя образа (тэг)> . - собать образ с именем
docker build . -f <имя файла>:<версия> - собрать образ по файлу

docker image inspect <имя или ид>:<версия> - просмотр инструкций выполнения образа

Содержит в себе: 
1.Пакеты программ
2.Файлы и конфигурации
3.Инструкции по сборке (последовательность и тд)


Структура dockerfile:
1.Базовый образ (image) - FROM ubuntu:22.04
2.Описание образа (опционально)- LABEL autor=RomNero
3.Команды, которые отработают внутри докер контейнера - RUN apt-get update
4.Рабочие директории - WORKDIR /var/www/html/
5.Файлы (копирование файлов на локальной машине в образ):
- COPY files2/script.sh /opt/script/sh - копирование в нужную папку
- COPY files2/index.html . - копирование в рабочую директорию
6.Работа с файлами - RUN chmod +x /opt/script/sh
7.Указание переменных - ENV TYPE=demo (если при запуске не были указаны переменные то будут указаны эти)
8.Порты - EXPOSE 80
9.Описание команд после запуска в контейнере:
- ENTRYPOINT ["echo"]
- CMD ["Hello my FIRST Docker"]
