Создать и запустить контейнер из выбранных образов
Если образа нет в локальном реестре, тогда он будет скачен с публичного репозитория и он попадет в локальный реестр
При запуске контейнеров у них должны быть разные порты
Если контейнер ничего не делает, тогда он закроется автоматически
Контейнеры с одинаковыми именами не могут быть созданы

docker run <аргументы> <image name или id> <команды запуска CMD> - создать контейнер из образов

Аргументы запуска:
--rm - удалить контейнер после его окончания работы
--name <имя> - задать имя кентейнеру
--network=<тип сети> - задать тип сети
-d - запустить в фоне, в случае с убунту будет доступен bash
-p <порт сервера:порт контейнера> - открыть порт у сервера к контейнеру для подключения к нему из вне
-e <имя переменной>=<значение переменной> - создать системную переменную внутри контейнера
-v <volumes> - монтирование диска в контейнере

Команды запуска: - команды, которые выполняются после запуска контейнера внутри контейнера
sleep <время> - дать задержку для закрытия образа
echo "text" - выдать текст после запуска контейнера
ls - отобразить рабочую дирректорию контейнера
